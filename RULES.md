# Vedic Reading 防污染规则

## 核心原则

对盘主进行 Vedic 占星分析时，**只能使用盘主提供的星盘数据**作为推理依据。禁止从以下来源获取信息并混入分析：

- CLAUDE.md / AGENTS.md（项目治理文件）
- 用户的 operator profile、tech stack、工作内容描述
- memory 目录中的任何记忆文件
- 对话中非星盘相关的上下文（用户的职业、兴趣、现有业务等）
- 用户的 git history、文件结构、项目列表

## 具体规则

### 1. 输入隔离

分析时视盘主为**完全陌生人**。除了星盘数据本身（行星位置、宫位、大运、Yoga、Shadbala 等），不使用任何关于盘主的背景信息。

### 2. 推导链条透明

每一个结论必须标注其推导来源。格式：

```
结论 ← 依据（必须是盘面数据）
```

如果一个结论无法追溯到具体的盘面数据点，则不应出现在分析中。

### 3. 具体 vs 抽象的边界

盘面**可以**推导的：
- 能力类型（Mercury 型 = 分析/沟通/交易）
- 能量强弱（Shadbala、SAV、Yoga）
- 时间窗口（大运/子限激活哪些宫位）
- 行为模式（行星关系、宫位互动）
- 身体区域对应（星座 → 身体部位）

盘面**不能**推导的：
- 具体行业（"做电商"、"做 AI"）
- 具体平台（"用 Shopee"、"用 Amazon"）
- 具体产品（"卖美妆"、"卖家居"）
- 具体公司/职位
- 具体人名/地名

### 4. 污染自检

输出分析前，对每一条推荐做以下检查：

> "如果我不知道这个人是谁、做什么工作、住在哪里，我是否仍然会给出这条推荐？"

如果答案是否 → 删除或改写为盘面层面的抽象描述。

### 5. 污染标注

如果分析中**不可避免**地使用了盘面以外的信息（例如用户主动提及自己的行业并要求针对性分析），必须明确标注：

```
[非盘面信息] 以下分析结合了盘主自述的 XX 背景，不是纯盘面推导。
```

## 违规示例

❌ "你适合做跨境电商" → 无法从盘面推导出"跨境电商"这四个字
✅ "Mercury AK + Venus AmK 合相事业宫，事业核心在智力与美学的交叉领域，适合交易+审美结合的方向" → 纯盘面

❌ "平台不限于 Shopee" → 泄露了已知盘主在做 Shopee
✅ "Rahu 第2宫大运支持非传统/数字化渠道" → 纯盘面

❌ "AI 工具创业" → 无法从盘面推导
✅ "Mercury 在自旺星座 + Bhadra Yoga = 擅长将复杂信息系统化为可用工具" → 纯盘面
