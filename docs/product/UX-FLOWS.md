# UX Flows v0.1

## 1. 首次免费排盘

`进入产品 → 填写出生资料 → 校验城市/时区 → Python排盘 → 轻量预览生成 → 免费命盘页`

输入：昵称、性别、出生日期、出生时间、出生城市、时间准确度。

用户不手填经纬度。系统解析并保存 timezone、latitude、longitude、altitude_m。时间不确定时明确提示对上升、宫位和细分判断的影响。

## 2. 免费命盘与分流

免费命盘首屏提供两个同等可见但有主次的路径：

- 主路径：`解锁完整报告`
- 次路径：`免费问盘`

免费用户不必购买报告即可进入 Agent。Agent 根据已保存的基础命盘回答，并在适当位置展示 `获取完整报告` 按钮。

## 3. 报告购买

`免费预览 → 点击解锁 → 确认当前人物和权益 → 支付 → 创建报告任务 → 章节进度 → 完整报告`

付款前显示：当前人物、章节范围、预计生成时间、附赠问盘次数、价格、免责声明。

预计时间必须由真实监控数据计算；测试前使用“预计数分钟”占位，不承诺固定时长。

## 4. 报告生成

真实状态：

1. 命盘数据验证
2. 核心结构分析
3. 事业与财富
4. 感情与关系
5. 身心健康
6. 大运与未来趋势
7. 一致性检查
8. 网页排版完成

允许离开页面。再次进入后恢复真实状态。付款成功后即使生成失败，订单权益仍保留，用户可重试或联系客服。

## 5. 章节追问

`报告章节 → 针对本章节继续提问 → 跳转Agent → 预填问题与章节上下文 → 显示消耗 → 用户确认发送 → 回答`

禁止点击章节按钮后直接扣除次数。只有真正提交问题时扣除；失败调用应自动退回或不入账。

## 6. 免费Agent

`免费命盘 → 免费问盘 → 选择人物 → 选择快捷问题或输入 → 确认消耗 → 回答`

回答结构：直接结论、盘面依据、时间背景、现实表现、建议、不确定性。

回答内的付费入口必须自然且相关，例如：

> 当前回答依据基础命盘与当前大运。完整报告会进一步覆盖事业结构、关系模式和未来时间窗口。

按钮：`查看完整报告内容`

## 7. 多人物档案

`档案 → 新建人物 → 填写出生资料 → 生成基础命盘 → 保存`

全局 Profile Switcher 出现在：命盘页、报告页、Agent页。切换人物时必须清空或切换相应上下文，绝不继续显示上一人的报告/对话。

## 8. 双人/多人问盘

`Agent → 分析对象选择器 → 单人/双人/多人 → 选择档案 → 选择关系类型 → 提问`

第一版建议完整实现单人和双人。多人先作为后续入口或实验功能。

双人关系类型：感情、婚姻、合作、一般关系。提问前明确显示预计消耗。

## 9. 分享与邀请

`命盘/报告 → 生成分享卡 → 选择模板 → 隐私检查 → 分享图片 → 新用户扫码 → 完成有效命盘 → 发放奖励`

分享卡不包含出生日期、时间、地点或可反推出个人身份的数据。二维码必须由真实邀请链接生成，不能使用概念图中的二维码。

## 10. 积分异常

- 次数不足：展示购买问盘包、邀请好友、返回报告三个选项。
- 请求失败：不扣除或自动退回。
- 重复提交：使用幂等键避免重复扣除。
- 退款：按规则收回尚未使用的赠送次数，已消费账目保留审计记录。
